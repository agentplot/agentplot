## Purpose

Miniflux client role providing restish-based CLI, skill delegation, API token management, and per-client base URL configuration via mkClientTooling.

## Requirements

### Requirement: Miniflux client role uses mkClientTooling with restish CLI
The miniflux client role SHALL be generated by `mkClientTooling` with a restish-based CLI package that bundles the Miniflux OpenAPI spec and configures API authentication via token header.

#### Scenario: Client CLI wraps restish with Miniflux API
- **WHEN** a miniflux client is configured with `name = "miniflux"` and `base_url = "https://rss.example.com"`
- **THEN** a CLI wrapper named "miniflux" SHALL be generated that invokes restish with the Miniflux OpenAPI spec loaded and `X-Auth-Token` header set from the client's API token secret

### Requirement: Miniflux client role bundles miniflux skill
The miniflux client role SHALL declare `capabilities.skills` with a SKILL.md file describing Miniflux operations via restish, including feed management, entry reading, category organization, and OPML import/export.

#### Scenario: Skill delegated to claude-code
- **WHEN** a miniflux client enables `claude-code.skill.enabled = true`
- **THEN** `programs.claude-code.skills` SHALL contain a miniflux skill entry with substituted client name

#### Scenario: Skill delegated to openclaw
- **WHEN** a miniflux client enables `openclaw.skill.enabled = true`
- **THEN** `programs.openclaw.skills` SHALL contain a miniflux skill entry

### Requirement: Miniflux client uses prompted API token secret
The miniflux client role SHALL use `capabilities.secret` with mode "prompted" for the Miniflux API token, generating a clan vars generator per client.

#### Scenario: API token secret generation
- **WHEN** a miniflux client "personal" is configured
- **THEN** a clan vars generator `agentplot-miniflux-personal-api-token` SHALL be created with a hidden prompt

### Requirement: Miniflux client exposes base_url as extraClientOption
The miniflux client role SHALL expose `base_url` via `extraClientOptions` for configuring the Miniflux instance URL per client.

#### Scenario: Multiple clients with different instances
- **WHEN** two miniflux clients are configured with different `base_url` values
- **THEN** each client's CLI wrapper SHALL use its respective base URL for API calls

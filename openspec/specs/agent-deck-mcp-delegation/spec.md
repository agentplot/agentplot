## MODIFIED Requirements

### Requirement: Single-client agent-deck MCP entry generation
When a client has `agent-deck.mcp.enabled = true`, the system SHALL generate a `programs.agent-deck.mcps.${clientName}` entry. The delegation logic SHALL be generated by `mkClientTooling` rather than hand-rolled per service. All services with MCP capability (linkding excluded — no MCP server, ogham-mcp, subcog, gno, qmd) SHALL support this target.

#### Scenario: Agent-deck MCP generated via mkClientTooling
- **WHEN** a service uses `mkClientTooling` with `capabilities.mcp` and a client enables `agent-deck.mcp.enabled = true`
- **THEN** `programs.agent-deck.mcps.${clientName}` SHALL contain the MCP config derived from `capabilities.mcp.urlTemplate`

### Requirement: Multi-client agent-deck MCP composition
When multiple clients across multiple services enable `agent-deck.mcp.enabled`, their MCP entries SHALL merge without conflict via the NixOS module system.

#### Scenario: Two services, each with one client enabling agent-deck MCP
- **WHEN** ogham client "memory" and qmd client "docs" both enable `agent-deck.mcp.enabled = true`
- **THEN** `programs.agent-deck.mcps` SHALL contain both `memory` and `docs` entries

## MODIFIED Requirements

### Requirement: qmd client role uses mkClientTooling
The qmd client role SHALL be generated by `mkClientTooling` with capabilities: `mcp = { type = "http"; urlTemplate = client: "https://${client.domain}/mcp"; }`, `skills = null` (qmd has no skill), `cli = null`. Extra client options: `domain`.

#### Scenario: qmd MCP-only client
- **WHEN** a qmd client enables `claude-code.mcp.enabled = true`
- **THEN** the interface SHALL contain only MCP-related targets (no skill, no CLI options)

#### Scenario: qmd multi-client
- **WHEN** two qmd clients are configured
- **THEN** each SHALL get independent HM modules with separate MCP entries

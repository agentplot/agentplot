## Why

Every agentplot clanService client role re-implements the same delegation options (claude-code skill/MCP, agent-skills, agent-deck MCP, openclaw skill) with copy-pasted wiring code. The 11 sub-agents that built our initial services each invented their own subset — linkding has 6 targets, ogham has 3, gno/qmd have 2, subcog has 2 with no profiles. There's no consistency, no way to add a new agent target without editing every service, and no reusable framework. The delegation pattern is agentplot's core value proposition and needs to be a first-class abstraction in agentplot-kit.

## What Changes

- **New `mkClientTooling` lib function in agentplot-kit**: A maximal helper that generates the entire client role (interface + perInstance) from a capabilities declaration. Services declare what they offer (skill, MCP, CLI, secret) and the function generates all target enable flags, the `clients = attrsOf clientSubmodule` interface, clan vars generators, HM module delegation, and agentplot.hmModules wiring.
- **Capabilities → targets model**: Services declare capabilities (`skill`, `mcp`, `cli`, `secret`). Targets (`claude-code`, `agent-skills`, `agent-deck`, `openclaw`) consume capabilities. Only targets whose required capabilities are provided generate enable flags. New targets can be added to agentplot-kit without changing any service.
- **agent-deck skill pool target**: Add `home.file.".agent-deck/skills/pool/${name}".source` wiring so service skills appear in agent-deck's Skills Manager TUI. Add corresponding `programs.agent-deck.skillSources` option to nix-agent-deck HM module for declarative pool management.
- **Multi-client everywhere**: All services (including gno, qmd, subcog) adopt the `clients = attrsOf clientSubmodule` pattern for named client configurations with independent target enablement.
- **Refactor linkding**: Migrate linkding's hand-rolled client role to use `mkClientTooling`.
- **Refactor ogham, subcog, gno, qmd**: Rewrite client roles to use `mkClientTooling` with consistent capability declarations.
- **Drop `claude-tools` target**: Remove `programs.claude-tools` delegation — it overlaps entirely with `programs.claude-code.skills` and `programs.agent-skills`. Remove `claude-plugins-nix` flake input.
- **Service README files**: Each service directory gets a user-focused `README.md` explaining what the service does, its roles, configuration options, and how to enable it in inventory.

## Capabilities

### New Capabilities
- `client-tooling-framework`: The `mkClientTooling` lib function in agentplot-kit — capabilities declaration, target registry, interface generation, delegation wiring, and secret management
- `agent-deck-skill-pool`: Declarative agent-deck skill pool population via HM module — nix-agent-deck `skillSources` option and agentplot wiring
- `service-readme`: User-focused README.md for each service directory

### Modified Capabilities
- `agent-deck-mcp-delegation`: Now generated by `mkClientTooling` instead of hand-rolled; multi-client pattern applied to all services
- `agent-skills-delegation`: Now generated by `mkClientTooling`; extended to all services with skill capability
- `openclaw-skill-delegation`: Now generated by `mkClientTooling`; extended to all services with skill capability
- `claude-code-mcp-delegation`: Now generated by `mkClientTooling`; multi-client pattern applied to all services
- `ogham-client`: Refactored to use `mkClientTooling` with multi-client pattern
- `subcog-client`: Refactored to use `mkClientTooling` with multi-client pattern
- `gno-client`: Refactored to use `mkClientTooling` with multi-client pattern
- `qmd-client`: Refactored to use `mkClientTooling` with multi-client pattern

## Impact

- **agentplot-kit**: New `lib/mkClientTooling.nix` added; `lib` output added to flake if not present
- **agentplot**: All 5 service client roles rewritten to use `mkClientTooling`; `claude-plugins-nix` flake input removed; `claude-tools` delegation code removed from linkding
- **nix-agent-deck**: New `programs.agent-deck.skillSources` HM option added for declarative skill pool management
- **Tests**: Existing delegation tests updated to reflect new structure; new test for `mkClientTooling` composition across multiple services
- **Inventory**: No breaking changes — existing enable flags remain the same, just generated differently. New `clients` wrapper needed for gno/qmd/subcog (currently flat options).

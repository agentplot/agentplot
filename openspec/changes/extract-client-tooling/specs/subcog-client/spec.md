## MODIFIED Requirements

### Requirement: subcog client role uses mkClientTooling
The subcog client role SHALL be generated by `mkClientTooling` with capabilities: `skill = ./skills/SKILL.md`, `mcp = { type = "http"; urlTemplate = client: "https://${client.domain}/mcp"; }`, `secret = { name = "jwt-secret"; mode = "generated"; }`. Extra client options: `domain`, `namespace`.

#### Scenario: subcog client with namespace isolation
- **WHEN** two subcog clients are configured with different namespaces
- **THEN** each SHALL get independent MCP entries that can target different namespace scopes

#### Scenario: subcog generated secret
- **WHEN** a subcog client is configured
- **THEN** a clan vars generator SHALL auto-generate the JWT secret via openssl

## MODIFIED Requirements

### Requirement: gno client role uses mkClientTooling
The gno client role SHALL be generated by `mkClientTooling` with capabilities: `mcp = { type = "http"; urlTemplate = client: "https://${client.domain}/mcp"; }`, `skill = null` (gno has no skill), `cli = null`. Extra client options: `domain`.

#### Scenario: gno MCP-only client
- **WHEN** a gno client enables `claude-code.mcp.enabled = true`
- **THEN** the interface SHALL contain only MCP-related targets (no skill, no CLI options)

#### Scenario: gno multi-client for different collection access
- **WHEN** two gno clients "docs" and "notes" are configured with the same domain
- **THEN** each SHALL get its own MCP entry under its client name

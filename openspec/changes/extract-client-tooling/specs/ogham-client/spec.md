## MODIFIED Requirements

### Requirement: ogham-mcp client role uses mkClientTooling
The ogham-mcp client role SHALL be generated by `mkClientTooling` with capabilities: `skills = [ ./skills/SKILL.md ]`, `mcp = { type = "sse"; urlTemplate = client: "https://${client.url}/sse"; }`, `secret = { name = "api-key"; mode = "prompted"; }`. Extra client options: `url` (SSE endpoint URL).

#### Scenario: ogham-mcp client with full targets
- **WHEN** an ogham-mcp client enables `claude-code.mcp.enabled`, `agent-deck.mcp.enabled`, and `claude-code.skill.enabled`
- **THEN** all three delegation targets SHALL be wired via mkClientTooling

#### Scenario: ogham-mcp multi-client for partition scoping
- **WHEN** two ogham-mcp clients are configured: "personal" and "work" with different names
- **THEN** each SHALL get independent HM modules with separate MCP entries and skill instances
